RaBbLE Documentation

v0.4.7 · RaBbLE-WebChat · April 2026

Technical reference for the RaBbLE frontend — a single-file HTML prototype for the RaBbLE a Boundless Behavioral Learning Engine interface. Covers every interactive element, its React state hookup, and how to connect it to a real AI backend.

What is RaBbLE

RaBbLE is a Collaborative Substrate — an AI-ambient environment that inhabits the machine alongside the user as a peer. The B in the acronym is a floating slot: Boundless is canonical, but Becoming, Brilliant, Behavioral are all valid states.

The frontend renders RaBbLE as a living entity: a particle nebula with two eyes (magenta left, cyan right), a dual-wave waveform mouth that responds to microphone input, and a theatrical boot sequence on first load.

The visual language is not decoration — it is character made visible. Every state change (idle → thinking → speaking) is expressed through the entity's appearance.

Architecture

The entire frontend lives in a single HTML file (RaBbLE.html). It uses React 18 + Babel (loaded from CDN) with all JSX inlined. No build step required.

Boot Sequence

On first render, booting state is true. The BootSequence component mounts as a fixed full-screen overlay at z-index: 9999.

The sequence runs on a timer cascade:

500ms → RaBbLE title fades in
850ms → Boot line 1 appears
1080ms → Boot line 2
1340ms → Boot line 3
1620ms → Boot line 4
1900ms → Boot line 5
2150ms → Boot line 6 (USER PROFILE: FOUND)
2500ms → Entity avatar fades in
4400ms → Overlay begins fade-out (opacity → 0)
5200ms → onDone() called → setBooting(false)

The main UI is always mounted but at opacity: 0 while booting. When setBooting(false) fires, it transitions to opacity: 1 over 0.6s — eliminating any pop between states.

Skip button

Positioned bottom-right of the boot overlay. Calls onDone() immediately.

Sidebar & Entity

The sidebar is a fixed-width (260px) column on the left. It contains the RaBbLE entity avatar, brand name, dynamic status, and navigation.

Entity Avatar — canvas renderer

The avatar is a <canvas> element rendered by RaBbLEAvatar. It runs a continuous requestAnimationFrame loop drawing three layers:

LayerDescriptionDriven by
Particles320 particles in a gaussian cluster. Blues, cyans, purples, whites. Drift with Lissajous phase.rabbleState
EyesTwo white oval orbs. Left = magenta outline, right = cyan. Each floats above/below its portal disc.mouse position
PortalsDark flat disc behind each eye. Left portal below orb. Right portal above orb. Fixed position.static
WaveformDual braided waves — magenta + cyan, phase offset by π. Only rendered when mic is active.micActive micLevel

Eye tracking

A mousemove listener on window populates mouseRef. Each frame, the orb X/Y is offset toward the cursor proportional to distance (capped at 9px X, 6px Y). Portals remain fixed. Effect: RaBbLE watches you.

Status pill

Displays current rabbleState: idle, thinking, or speaking. The dot color changes: cyan for idle/speaking, violet for thinking. The "B" word in the subtitle also shifts: Boundless / Becoming / Brilliant.

Chat (Convo view)

Messages are stored in the msgs array. Each message has a role ('rabble', 'user', 'system') which controls alignment and color. Messages drift in via CSS msgDrift keyframe.

When a message is sent, RaBbLE enters thinking state for 1800ms, then transitions to speaking and appends a reply, then returns to idle after 3000ms. Replace the setTimeout mock with a real window.claude call to get live responses.

State Hookups

All global state lives in the App component and flows down via props.

// Global state (App)
const [booting, setBooting] // boot overlay visible
const [screen, setScreen] // active screen ID
const [contentMode, setContentMode] // 'convo' | 'renders'
const [rabbleState, setRabbleState] // 'idle' | 'thinking' | 'speaking'
const [msgs, setMsgs] // message array
const [input, setInput] // text input value
const [micActive, setMicActive] // mic enabled
const [micLevel, setMicLevel] // 0–1 amplitude from analyser
const [tweaks, setTweak] // design tweaks (cardVariant, etc.)

rabbleState flow

User sends message
→ setRabbleState('thinking') // eyes shift violet, particles accelerate
→ await AI response (1800ms mock / real claude call)
→ setRabbleState('speaking') // waveform amplitude up
→ append reply to msgs
→ setRabbleState('idle') // 3000ms later

Wiring Real AI

The mock response in sendMsg uses a setTimeout. Replace it with a window.claude.complete() call (available in-environment) or a direct Anthropic API call for local use. 

// Current mock (in sendMsg):
setTimeout(() => {
  setRabbleState('speaking');
  setMsgs(m => [...m, { id: uid+1, role: 'rabble', text: "…" }]);
  setTimeout(() => setRabbleState('idle'), 3000);
}, 1800);

// Replace with (in-environment):
const reply = await window.claude.complete({
  messages: [
    { role: 'system', content: 'You are RaBbLE, a Boundless Behavioral Learning Engine…' },
    ...msgs.map(m => ({ role: m.role === 'user' ? 'user' : 'assistant', content: m.text })),
    { role: 'user', content: input }
  ]
});
setMsgs(m => [...m, { id: uid+1, role: 'rabble', text: reply }]);
setRabbleState('speaking');
setTimeout(() => setRabbleState('idle'), 3000);
For local use outside this environment, replace window.claude.complete with a fetch call to https://api.anthropic.com/v1/messages with your API key in the header. Never hardcode API keys in client-side HTML for production.

Mic & Audio

The mic button calls navigator.mediaDevices.getUserMedia({audio:true}). On success, it creates a Web Audio AnalyserNode, reads getByteFrequencyData every frame, and sets micLevel (0–1 normalised) which drives waveform amplitude in the avatar canvas.

Mic button click
→ getUserMedia({ audio: true })
→ AudioContext → MediaStreamSource → AnalyserNode
→ rAF loop → getByteFrequencyData → avg / 128
→ setMicLevel(0–1)
→ RaBbLEAvatar: drawWaveform amplitude = base + level × 18

The waveform is only rendered when micActive is true. When mic is off, drawWaveform is not called and the mouth is invisible.

Artifact Renderer

Current artifact cards contain static mock content (chart, code, analysis text). To render live applets, replace the card-body content with a sandboxed <iframe>: 

// Inside a card's body:
<iframe
  src="data:text/html;charset=utf-8,…"
  sandbox="allow-scripts"
  style="width:100%;height:200px;border:none;border-radius:8px"
/>

Pass the artifact's HTML content as a data: URL or srcdoc attribute. The sandbox="allow-scripts" attribute lets JavaScript run inside the frame while blocking navigation and cross-origin access.
